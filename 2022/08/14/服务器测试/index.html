服务器测试

发表于 2022-08-14 分类于 专业知识

与人交往要守住态度，值得的我一定真心相待不辜负，不值得的一笑而过不再多说。要做一个有原则的人，不亏待每一份热情，也绝不讨好任何的冷漠。

  1. 物理服务器配置检查
    1. 查看处理器信息：cat /proc/cpuinfo(关注 cpu 的核心数量，频率，支持的功能开启状态，超线程)
    1. 查看内存信息：cat /proc/meminfo（大小，频率，品牌）
    1. 查看内存条查法顺序：dmidecode 查看 memory（可查看每个内存槽位是否有内存条）
    1. 查看硬盘信息：fdisk -l/lsblk（硬盘分区详细情况，硬盘分布）
    1. 查看电源信息：从服务器背面电源位置进行查看
    1. 查看网卡信息，输入命令 lspci | grep Ethernet
    1. 查看网口信息，ifconfig -a/ethtool eth0
    1. 查看主板信息，dmidecode -t bios（关注固件版本）

  1. 物理服务器各部件的作用和指示灯的含义

    一般绿灯正常运行，黄灯存在警告，红灯存在故障

  2. 电源、风扇和硬盘的冗余性检查

    通过手动直接拔出硬件查看是否存在冗余，硬盘除 raid0 和直通其余类型均可冗余（但冗余存在限制）

  3. ipmi 功能验证

      1. 验证服务器 ipmi 具备开机、关机、重启、设置 PXE 引导、查看服务器健康状态、查看日志等功能（ipmi -I lanplus -H 192.168.5.7 -U zteroot -P Superuser9! chassis power on/power off/power cycle/bootdev pxe/sdr/sel list/clear）

  4. 服务器 BMC 功能检查

  • 检查并登记所有送测服务器的 BMC 版本
  • BMC 检查服务器状态和查看报错日志的方法
  • 通过操作系统命令，查看 BMC 版本（Firmware Revision）,查看序列号，确认 BMC 和 BIOS 中的序列号已知
  • 验证通过操作系统命令，查看和设置 BMC 参数命令
  1. BMC 管理
  2. BMC 对整机的故障管理和 FW 升级
  3. BIOS 快捷键测试
  4. BIOS 界面功能检查
  5. 利用系统命令进行服务器 BIOS 设置和检查
  • dmidecode -t bios #查看 BIOS 版本
  • cpupower idle-info #检查 Cstate 开关
  • cat /proc/cpuinfo | grep “sibling” | uniq
  • cat /proc/cpuinfo | grep “cpu cores” | uniq #检查 HT 超线程技术开关
  • cat /proc/cpuinfo | grep vmx #查看 cpu 虚拟化功能
  1. HTML5 KVM 功能测试
  2. 通过 ISO 光盘安装操作系统
  3. IPV6 功能测试
  4. 服务器 Redfish 测试
  5. Kdump 生成测试
  6. 主板晶振检测
  7. 电源模块高可用测试
  8. 双网卡绑定的高可用测试
    1. 在操作系统中将制定网卡进行双网卡绑定，绑定为负载均衡 xor 模式
    1. 在操作系统中发起到交换机网关的 ping
    1. 操作系统中打开 nmon，持续观察网络流量
    1. 在服务器端拔掉主网卡网线，观察 ping 的丢包情况
    1. 插回主网线，观察 ping 的丢包情况，查看 nmon 中流量是否回到主网卡
    1. 在服务器端拔掉备网卡网线，观察 ping 的丢包情况
    1. 插回备网线，观察 ping 的丢包情况
    1. 重复上述步骤 2-7 次，查看丢包情况
  1. 服务器综合性能打分
  • 通过 Unixbench 为服务器进行整体打分
  1. 网络带宽性能测试
    1. 机器上安装 netperf server 端和 client 端
    1. 配置双网卡绑定负载均衡模式
    1. 发起 netperf 网络压力，观察 nmon 中的网卡流量
  1. 服务器耗电量测试
    1. 在主机上使用工具发起压力测试，从 25%-100%
  • cpu：stress -c CPU 核数
  • 内存：Memtester
  • 硬盘：dd if=/dev/sdx of=/dev/zero
    1. 通过 ipmi 读出当前服务器用电量
  1. 服务器 cpu 性能测试
  • 使用 spec2017 软件
  1. 服务管理
  • 启动服务 systemctl start sshd service
  • 关闭服务 systemctl stop sshd service
  • 重启服务 systemctl restart sshd service
  • 重新加载服务配置文件 systemctl reload sshd service
  • 检查服务运作状态 systemctl status sshd service/is-active
  • 展开所有服务状态详细信息 systemctl list-units –type service –all
  • 允许服务开机启动 systemctl enable sshd service
  • 禁止服务开机启动 systemctl disable sshd service
  1. 网络配置测试
  • 在/etc/sysconfig/network-script 目录下
  • nmcli con add type bridge con-name brl ifname brl
  • nmcli con mod brl ipv4 addresses 192.168.3.88
  • nmcli con mod brl ipv4 method manual
  • nmcli con add type bridge-slave con-name brl-port0 ifname ens6f0 master brl
